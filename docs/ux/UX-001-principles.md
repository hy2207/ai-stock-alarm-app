# UX-001: 제품 UX 원칙 · 페르소나 · JTBD

> 이 문서는 PRD §1/§2, ADR-001~005, SRS §2.1을 기반으로 이후 화면 설계(UX-002~016)가 따라야 할 UX 기준을 정의한다.

## 제품 UX 비전

사용자가 미국주식 정보를 오래 탐색하지 않고, 관심종목 기준으로 **오늘 살지, 팔지, 기다릴지**를 3분 안에 판단할 수 있게 한다. 출력물은 **실행 가능한 숫자**(방향, 진입가/범위, 목표가/범위, 보유기간, 신뢰도 수준)를 담은 1~3장의 **의사결정 카드**이며, 뉴스 요약이나 차트가 아니다.

## 페르소나

### 페르소나 A: 지훈 — 준경험 직장인 투자자

| 항목 | 내용 |
|---|---|
| 직업 | 10년 차 IT 기획자, 30대 후반 |
| 투자 경험 | 3~5년 미국 개별주 매매 경험, 기술적 분석 용어는 알지만 매일 차트를 볼 시간이 없음 |
| 하루 투자 시간 | 출근 전 10~15분 |
| Pain Point | "뉴스는 계속 쌓이는데 결국 내가 매수/매도해야 할 가격을 못 찾겠다. 오늘 AAPL을 살지 말지만 알면 된다." |
| 이 제품이 해결하는 방식 | 아침에 푸시 또는 홈 진입 → 1~3장 카드 확인 → 방향·가격·기간을 보고 실행 판단. 추가 리서치 불필요. |

### 페르소나 B: 미영 — 불신형 유료 정보 구독자

| 항목 | 내용 |
|---|---|
| 직업 | 15년 차 마케팅 매니저, 40대 초반 |
| 투자 경험 | 유료 뉴스레터 3개 구독, 종목 추천 서비스 이용 경험 있음 |
| 핵심 불신 | "추천은 성과 좋은 것만 보여주고 실패는 숨긴다. 결국 광고 수익 모델 아니냐." |
| Pain Point | 정보는 넘치지만 **누구를 믿을지**를 판단하는 데 에너지를 씀. "왜 이 종목을 추천하는지, 이전 추천은 얼마나 맞았는지"를 확인해야 납득함. |
| 이 제품이 해결하는 방식 | 모든 카드에 **한 줄 이유**(reasonLine)와 과거 성공/실패 기록(Trust Layer)이 함께 노출되어 블랙박스 느낌이 없음. |

## Jobs To Be Done

1. **"아침 3분 안에 오늘의 매수/매도 대상을 결정해줘."** — 출근 전 빠른 판단이 필요할 때, 종목별 뉴스를 읽지 않고 카드로 의사결정.
2. **"리스크 성향에 따라 추천이 어떻게 달라지는지 보고 싶어."** — 공격형/중립형/안정형 모드를 변경하며 각 모드가 제시하는 카드의 방향과 가격을 비교.
3. **"이 서비스의 추천은 과연 믿을 만한지 확인하고 싶어."** — 카드에 표시된 추천의 근거와 이전 성과(성공/실패)를 검토.
4. **"바쁜데 놓치는 종목 없게 아침 브리핑을 받아보고 싶어."** — 푸시 알림으로 핵심 추천을 확인, 딥링크로 앱 내 해당 카드로 즉시 이동.
5. **"오늘은 시간이 있으니 좀 더 자세히 들여다볼게."** — 카드를 클릭해 상세 정보(유사 패턴, 섹터 맥락)를 확인.

## UX 원칙

### P1. Decision Layer 우선 (ADR-001)
모든 화면의 1차 출력은 **실행 가능한 의사결정 카드**다. 시장 뉴스, 차트, 데이터표는 의사결정을 보조하는 2차 정보로만 존재한다.

### P2. Confidence는 인터랙션 (ADR-002)
사용자가 선택한 리스크 모드(공격형/중립형/안정형)는 단순한 표시값이 아니라 **카드의 방향과 가격에 직접 영향을 미치는 선택 도구**다. 3개 모드의 카드는 한 번의 LLM 호출로 미리 생성되어 전환이 즉시 이루어진다.

### P3. 결과 우선, 지표는 뒤에 (ADR-004)
카드 상세의 메인 폴드에는 방향·진입가·목표가·보유기간·신뢰도·한 줄 이유가 표시된다. 캔들 차트, RSI, MACD 등 기술적 지표는 메인 폴드에 배치하지 않는다.

### P4. 투명성으로 신뢰 구축 (ADR-005)
모든 카드는 생성 근거 한 줄(reasonLine)과 함께, 해당 서비스의 과거 추천 성과(성공뿐 아니라 실패 포함)에 접근할 수 있는 Trust Layer를 제공한다. "데이터 축적 중" 상태도 숨기지 않고 표시한다.

### P5. 3분 이내 판단 완료
홈 화면 진입부터 1~3개 카드 검토, 의사결정까지 3분 이내에 끝낼 수 있어야 한다. 불필요한 로딩, 과도한 스크롤, 여러 단계의 클릭을 방지한다.

### P6. 점진적 공개 (Progressive Disclosure)
첫 화면에서는 핵심 결정 정보(방향, 가격, 보유기간)만 노출한다. 사용자가 더 알고 싶을 때 상세 화면으로 이동해 근거, 유사 패턴, 성과 기록을 확인할 수 있다.

### P7. 상태 드러내기 (No Hidden Errors)
데이터 부족, LLM 생성 실패, 네트워크 오류 상황에서도 사용자에게 빈 화면이나 5xx 페이지를 노출하지 않는다. No Call 카드, "데이터 축적 중" 메시지 등 명확한 안내로 상태를 전달한다.

## Non-Negotiables

다음은 ADR 및 기능 요구사항에서 파생된 UX 불가침 규칙이다.

- 카드 메인 폴드에 캔들/RSI/MACD 차트를 배치하지 않는다. (ADR-004)
- Confidence Score는 읽기 전용 배지가 아닌 3단계 선택 인터랙션이다. (ADR-002)
- 추천 카드는 사용자당 최대 3개를 초과하지 않는다. (REQ-FUNC-010)
- 모든 카드는 "투자 참고용 정보이며 투자 자문이 아님" 면책 문구를 고정 표시한다. (REQ-FUNC-085)
- Trust Layer는 성공 기록만 선택적으로 표시하지 않으며, 실패 기록도 동등하게 노출한다. (ADR-005)
- 오류/데이터 부족 상황에서 HTTP 5xx를 사용자에게 직접 노출하지 않는다. (REQ-FUNC-083)
- `/api/events` 엔드포인트를 구현하지 않으며, 행동 이벤트는 PostHog SDK로 직접 전송한다. (SRS v0.3)

## 경험 필러: 핵심 사용자 흐름

### 아침 브리핑 (Push → 홈 → 결정)
1. 사용자가 아침 푸시 알림을 수신 → 탭 → 앱 진입
2. 홈 화면에 1~3개 추천 카드 로딩 (데이터 부족 시 No Call 카드)
3. 카드의 방향/가격/기간 확인 → 리스크 모드 변경하며 비교
4. 결정 또는 상세 확인

### 셀프 체크 (앱 진입 → Confidence 변경 → 카드 검토)
1. 사용자가 직접 앱 실행 → 홈 화면
2. 저장된 riskMode가 기본 선택됨 (변경 가능)
3. Confidence Score 변경 시 LLM 재호출 없이 기 생성된 3벌 카드 중 해당 모드 카드 표시
4. 카드 하단 면책 문구 확인 → 종목 선택해 상세 이동

### Trust 검토 (성과 기록 → 신뢰 평가)
1. 사용자가 이전 추천의 성과를 확인하려 함 → 이력 화면 접근
2. 종목별 과거 추천 목록에서 성공/실패/평가 중 상태 확인
3. realizedReturn(실제 수익률)로 정량 평가
4. "데이터 축적 중" 상태에서는 숨기지 않고 해당 상태 표시
