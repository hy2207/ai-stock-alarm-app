# 예측 모델을 두 번 갈아엎은 이유: Conformal Prediction과 Yang-Zhang 변동성 도입기

지금까지 다섯 편은 시장조사부터 서브에이전트 오케스트레이션까지, 프로세스와 하네스 얘기였다. 이번 편은 조금 다르다 — 실제로 짠 알고리즘 하나가 예상보다 훨씬 부끄러운 성적표를 받았고, 그걸 세 단계에 걸쳐 갈아엎은 과정을 그대로 기록한다. 지금도 이 통계 예측 모듈은 계속 손보고 있는 중이라, 회고이면서 동시에 진행 중인 이야기다.

이 프로젝트에는 AI(Gemini) 판단과 완전히 독립적으로 돌아가는 통계 예측 모듈이 있다. 외부 라이브러리 없이 순수 TypeScript로 짠 시계열 모델(`src/lib/quant/`)이 보유 기간 후 예상가와 범위를 계산해서 카드에 얹어준다. 처음 버전을 실측 검증했을 때 나온 숫자를 보고 솔직히 당황했다.

## 1. Version 1 — 교과서대로 짰는데 랜덤워크보다 못했다

첫 버전은 이론적으로는 무난한 조합이었다.

- 예상가: Holt's Double Exponential Smoothing(α=0.5, β=0.3) + Linear Regression 추세 외삽의 ensemble 평균 (최근 21거래일)
- 예상 범위: 일별 log return의 EWMA 변동성(λ=0.94, RiskMetrics) × √(보유일수), ±1σ (이론 커버리지 68%)

M7(AAPL·MSFT·GOOGL·AMZN·NVDA·META·TSLA) 39건을 walk-forward로 채점해봤다. 결과는:

- 범위 적중률 **31%** — 이론값 68%에 한참 못 미침
- 평균 오차 3.52% 중 **+2.90%가 방향 편향** — 하락장에서 맞춘 추세를 반등장에서도 계속 아래로 외삽
- 추세를 100% 반영한 예측이 **random walk 기준선(그냥 전일 종가 그대로, 오차 2.23%)보다도 부정확**(3.25%)

세 번째가 제일 뼈아팠다. "아무 모델도 안 쓰고 어제 종가를 그대로 오늘 예측으로 쓰는" 극단적으로 단순한 기준선보다 내가 짠 모델이 더 틀렸다는 뜻이다. 일별 주가에서 추세(drift)를 추정하려는 시도 자체가 대부분 노이즈를 학습한다는, 통계학에서 이미 잘 알려진 얘기를 직접 데이터로 확인한 셈이다.

## 2. 3단계로 나눠서 고치기로 했다

문제를 한 번에 다 고치려고 하지 않고, 커밋을 3단계로 쪼갰다. 각 단계마다 같은 M7 데이터로 walk-forward 재검증을 거쳤다.

**1단계 — Conformal Prediction으로 밴드부터 고치기**

가장 먼저 손댄 건 범위 적중률(31%)이었다. ±1σ 정규분포 가정이 fat tail과 국면 전환을 과소평가한다고 보고, 밴드 폭 자체를 "이 종목의 최근 실제 예측 오차 분포"에서 직접 뽑아오는 방식(conformal prediction)으로 바꿨다. 종목의 walk-forward 1일 오차 분위수(80% 목표)를 √(보유일수)로 스케일링하는 방식이다. 분포를 가정하지 않으니 종목마다 다른 변동성 패턴에 자동으로 맞춰진다. 이 단계만으로 적중률이 31% → 60%로 올라갔다.

**2단계 — Yang-Zhang OHLC 변동성으로 표본 효율 올리기**

conformal 밴드에도 최소한의 σ 하한(floor)은 필요한데, 여기 쓰던 EWMA가 종가만 사용하는 방식이라 20~60일짜리 짧은 윈도우에서 표본 효율이 떨어졌다. Yang-Zhang 추정량은 오버나이트 갭, 시가-종가, Rogers-Satchell 구성요소를 결합해서 OHLC 네 값을 전부 쓰는 range 기반 추정이다. close-to-close 대비 표본 효율이 약 8배 높다는 게 알려져 있고, 마침 `TickerPriceHistory`에 OHLC를 이미 저장해두고 있어서 데이터를 새로 모을 필요도 없었다.

**3단계 — Damped Trend + Random Walk Shrinkage로 방향 편향 잡기**

밴드는 나아졌는데 점 예측의 방향 편향(+2.90%)은 그대로 남아있었다. 여기서 두 가지를 같이 적용했다.

- Holt를 Gardner-McKenzie의 damped trend(φ=0.85)로 바꿔서, 추세를 끝까지 밀지 않고 감쇠시킨다. Linear Regression 쪽도 같은 damped horizon으로 외삽해서 앙상블 두 멤버가 같은 철학을 공유하게 만들었다.
- 그래도 남는 편향은 결국 추세 신호 자체가 노이즈라는 뜻이라, 예측을 전일 종가 쪽으로 수축(shrinkage)시켰다. 저장된 M7 이력에 가중치를 스윕해보니 추세 가중치를 낮출수록 단조롭게 좋아졌다 — w=1.0(추세 100%)일 때 3.25%였던 오차가 w=0.3에서 2.33%까지 내려갔다. 그래서 최종 가중치를 0.3으로 고정했다.

## 3. 실측 결과: 랜덤워크를 겨우 이겼다

같은 M7 데이터로 다시 walk-forward 채점한 결과다.

| 지표 | V1 | V2 |
|---|---|---|
| 범위 적중률 (목표 80%) | 31% | **85%** |
| 평균 절대 오차 | 3.52% | **2.38%** (random walk 하한 2.23%) |
| 방향 편향 | +2.90% | **+1.30%** |

여기서 강조하고 싶은 건 "완전히 이겼다"가 아니라는 점이다. V2도 random walk 기준선(2.23%)을 아주 살짝 이기는 수준(2.38%)이다. 일별 주가 예측에서 순수 통계 모델이 낼 수 있는 현실적인 개선폭이 이 정도라는 걸 그대로 받아들이기로 했다. 대신 ARIMA나 LSTM처럼 더 복잡한 모델은 의도적으로 계속 배제하고 있다 — 하루 단위 20~60개 포인트로는 오버피팅 위험이 크고, serverless 환경에서 학습 비용을 감당할 이유도 없다. 지금 조합은 외부 라이브러리 없이 밀리초 단위로 계산되면서 각 구성요소를 설명할 수 있다는 점이 오히려 이 프로젝트의 제약(ADR-004: 지표를 신비화하지 않는다)과 더 잘 맞는다.

## 4. 지금도 계속 손보고 있는 부분

이 글을 쓰는 지금도 이 모듈은 완결된 상태가 아니다. 진행 중인 것들을 정리하면:

- **최소 데이터 기준선 재검토**: 지금은 유효 종가 20거래일 미만이면 예측 자체를 생략(null)하는데, 이 임계값이 20이 맞는지 아직 확신이 없다. IPO 직후 종목이나 최근 상장주에서 이 기준이 너무 보수적인지 계속 관찰 중이다.
- **σ floor와 conformal 밴드의 상호작용**: conformal 분위수가 floor보다 좁아지지 않도록만 설계해뒀는데(소표본에서 과소평가 방지), 반대로 conformal이 과도하게 넓어지는 케이스가 있는지는 아직 M7 7종목 데이터만으로는 표본이 부족해서 관심종목 확장 데이터가 쌓이는 대로 다시 봐야 한다.
- **추세 가중치 0.3의 일반화 여부**: 지금 w=0.3은 M7 히스토리 스윕에서 나온 값인데, 이게 M7 대형주 특유의 낮은 변동성 패턴에 과적합된 값인지, 아니면 더 넓은 유니버스에서도 안정적인지는 아직 검증 전이다.

이번 편은 결론을 깔끔하게 못 낸다는 게 오히려 솔직한 결론이다. 시장조사, PRD, SRS, 태스크 자동화, 멀티 에이전트 하네스까지는 한 번 세워두면 비교적 안정적으로 굴러가는 구조였는데, 정량 모델은 실측 데이터가 쌓일 때마다 계속 다시 검증하고 고쳐야 하는 영역이다. 이 시리즈는 여기서 일단 마무리하지만, 이 예측 모듈 얘기는 데이터가 더 쌓이면 후속 편으로 다시 다룰 것 같다.
