# Figma Make 구현 프롬프트: AI Stock Alarm App

## 목표

미국 주식 투자자를 위한 **리스크 맞춤형 의사결정 인터페이스**를 구현해줘.

이 앱은 일반적인 뉴스 요약 앱이나 차트 앱이 아니다. 핵심은 사용자가 복잡한 차트와 지표를 보지 않고도 **오늘 무엇을, 어떤 방향으로, 얼마에, 며칠 동안 볼지** 빠르게 판단하게 만드는 것이다.

제품의 핵심 컨셉은 다음과 같다.

- 정보 제공 서비스가 아니라 **Decision Layer**
- 차트 중심 UI가 아니라 **결과 중심 UI**
- AI 추천을 일방적으로 보여주는 것이 아니라 **사용자가 리스크 성향을 직접 선택**
- AI 결과를 블랙박스로 만들지 않고 **한 줄 이유, 성과 이력, 실패 기록을 포함한 Trust Layer** 제공
- 사용자는 한국 거주 미국주식 투자자이며, 장중 대응이 어려운 바쁜 직장인이다.

## 구현 범위

반응형 웹 앱 프로토타입을 만들어줘. 실제 백엔드는 필요 없고, 모든 데이터는 프론트엔드 mock data로 처리한다.

다음 화면을 구현해줘.

1. 로그인 화면
2. 온보딩 관심 종목/섹터 선택 화면
3. 홈 추천 카드 화면
4. 추천 상세 화면
5. 추천 이력 아카이브 화면
6. 설정 화면
7. 푸시 딥링크 랜딩/상태 화면
8. No Call, Loading, Empty, Error 상태

## 기술/스타일 기준

- Next.js 앱처럼 보이는 모바일 우선 반응형 웹 UI로 구현
- Tailwind CSS와 shadcn/ui 기반 느낌
- 실제 구현은 Figma Make 안에서 동작하는 프로토타입이면 됨
- 카드, 버튼, 탭, segmented control, alert, table/list, switch를 사용
- 캔들 차트, RSI, MACD, 복잡한 원본 차트 위젯은 절대 넣지 말 것
- 숫자, 방향, 가격, 기간, 이유, 성과 이력을 중심으로 디자인
- 메인 화면은 모바일에서 390px 폭 기준으로 자연스럽게 보여야 하고, 데스크톱에서는 최대 1120px 콘텐츠 영역으로 확장

## 디자인 방향

전문 금융 앱처럼 신뢰감 있게 만들되, 차갑고 복잡한 트레이딩 터미널처럼 보이지 않게 해줘.

시각 톤:

- 배경: 아주 밝은 회색 또는 off-white
- 기본 텍스트: 거의 검정에 가까운 slate 계열
- 보조 텍스트: muted gray
- 상승/BUY: green 계열
- 하락/SELL: red 계열
- 중립/No Call: gray 또는 amber 계열
- 주요 CTA: 신뢰감 있는 blue 계열
- 카드 radius는 8px 이하
- 카드 내부는 정보 밀도를 높이되 여백은 충분히 둠
- 그림자보다 border와 배경 대비로 계층을 표현
- 과도한 그라데이션, 장식용 blob, 과한 일러스트는 사용하지 말 것

## 전역 내비게이션

앱 하단 또는 상단에 다음 내비게이션을 제공해줘.

## 화면 상세 요구사항

### 1. 로그인 화면

목적: 미인증 사용자가 로그인해야 하는 이유와 다음 행동을 명확히 이해하게 한다.

구성:

- 앱 이름: AI Stock Alarm
- 헤드라인: 아침 3분, 오늘의 미국주식 판단
- 서브카피: 차트 없이 방향, 가격, 보유 기간, 근거를 한 장으로 확인하세요.
- Google로 계속하기 버튼
- Kakao로 계속하기 버튼
- 이메일로 계속하기 입력 영역
- 보호 경로에서 온 경우 안내 문구:
  - 추천 카드와 관심 종목 설정을 보려면 로그인이 필요합니다.
- 하단 면책:
  - 투자 참고용 정보이며 투자 자문이 아닙니다.

상호작용:

- 로그인 버튼 클릭 시 온보딩 화면으로 이동
- 이미 온보딩 완료 상태를 시뮬레이션할 수 있도록 "데모로 홈 보기" 보조 버튼 제공

### 2. 온보딩 화면

목적: 사용자가 관심 종목 또는 섹터를 최소 1개, 최대 3개 선택하게 한다.

구성:

- 헤드라인: 관심 있는 종목을 3개까지 선택하세요
- 선택 카운터: 2/3 선택됨
- ticker/sector 선택 chip grid
- 선택된 항목은 명확한 selected 상태
- 3개 초과 선택 시 toast 또는 inline alert:
  - 최대 3개까지 선택 가능합니다.
- 0개 선택 상태에서는 CTA disabled
- CTA:
  - 선택 완료
- 보조 카피:
  - 선택한 종목은 아침 추천 카드 생성에 사용됩니다.

상호작용:

- chip 선택/해제 가능
- 1개 이상 선택 시 CTA 활성화
- CTA 클릭 시 홈 화면으로 이동

### 3. 홈 추천 카드 화면

목적: 사용자가 홈 진입 후 3분 이내에 실행 후보를 판단하게 한다.

구성:

- 상단:
  - 오늘 날짜
  - "오늘의 의사결정 카드"
  - "관심 종목 기준 3개 이하로 압축했습니다"
- Confidence Score segmented control:
  - 안정형
  - 중립형
  - 공격형
- 선택 변경 시 카드의 가격, 보유기간, actionLabel 중 최소 하나가 즉시 바뀌어야 함
- 추천 카드 1~3개 표시
- 각 카드 필수 정보:
  - ticker
  - company name
  - BUY/SELL 방향 badge
  - actionLabel
  - 진입가 또는 진입 범위
  - 목표가 또는 목표 범위
  - 손절 기준가
  - 보유 기간
  - confidenceScore
  - 한 줄 이유 reasonLine
- CTA:
  - 가격 복사
  - 상세 보기
  - 브로커로 이동
- 카드 하단 고정 면책:
  - 투자 참고용 정보이며 투자 자문이 아닙니다.

카드 디자인:

- ticker와 방향을 가장 먼저 보이게
- 가격 정보는 큰 숫자로 표현
- 보유 기간은 "권장 보유 5일"처럼 명확하게
- 한 줄 이유는 160자 이하로 보이게
- 상세 보기 버튼은 primary 또는 secondary로 명확히

상호작용:

- Confidence Score 변경 시 모든 카드가 해당 risk mode 값으로 즉시 전환
- 가격 복사 클릭 시 작은 toast:
  - 가격이 복사되었습니다.
- 브로커로 이동 클릭 시 confirmation modal:
  - 외부 브로커 화면으로 이동합니다.
- 상세 보기 클릭 시 추천 상세 화면으로 이동

### 4. 추천 상세 화면

목적: 홈 카드의 핵심 정보를 유지하면서 더 많은 근거와 성과 이력을 보여준다.

절대 금지:

- 캔들 차트
- RSI
- MACD
- 복잡한 트레이딩 차트 위젯
- 과도한 기술 지표 패널

구성:

- 뒤로가기
- ticker, company name
- BUY/SELL 방향
- actionLabel
- 가격 요약 영역:
  - 진입가/진입 범위
  - 목표가/목표 범위
  - 손절가
  - 보유 기간
- Confidence Score control 유지
- 한 줄 이유 카드
- 근거 스냅샷:
  - 뉴스 신호
  - 거래량 신호
  - 커뮤니티 신호
  - 패턴 태그
- 성과 카드:
  - 최근 30건 기준 성공률
  - 평균 수익률
  - 실패 건수
  - 평가 중 건수
- 성과 이력 리스트:
  - 성공/실패 모두 표시
  - realizedReturn 표시
  - evaluatedAt 표시
- 유사 패턴 섹션:
  - 데이터가 있으면 표시
  - 없으면 조용히 숨김
- CTA:
  - 가격 복사
  - 알림 설정
  - 브로커로 이동

Trust Layer 표현:

- 실패 기록을 부끄럽게 숨기지 말 것
- "실패 포함 기록"이라는 표현을 사용해 신뢰감 있게 보여줄 것
- 데이터 부족 시 빈 표 대신:
  - 데이터 축적 중
  - 이 종목은 아직 충분한 평가 기록이 없습니다.

### 5. 추천 이력 아카이브 화면

목적: 종목별 과거 추천 결과를 최신순으로 확인하게 한다.

구성:

- 헤드라인: 추천 이력
- 필터:
  - 전체
  - AAPL
  - TSLA
  - NVDA
- 정렬:
  - 최신순 기본
- 리스트 필드:
  - ticker
  - predictedDirection
  - realizedReturn
  - hitFlag
  - evaluatedAt
- 성공은 green, 실패는 red 또는 neutral red
- 평가 중은 gray
- 빈 상태:
  - 아직 쌓인 추천 이력이 없습니다.
  - 관심 종목을 선택하면 추천 기록이 여기에 쌓입니다.
- CTA:
  - 관심 종목 수정하기

### 6. 설정 화면

목적: 관심 종목, 리스크 성향, 푸시 권한, 계정 상태를 관리하게 한다.

구성:

- 프로필 요약:
  - 화영
  - Asia/Seoul
- 관심 종목 설정:
  - 현재 선택된 AAPL, TSLA, NVDA
  - 최대 3개까지 수정 가능
  - 저장 버튼
  - 저장 완료 feedback:
    - 변경사항이 저장되었습니다. 다음 추천부터 반영됩니다.
- 기본 리스크 성향:
  - 안정형 / 중립형 / 공격형 segmented control
- 푸시 수신:
  - 아침 브리핑 받기 switch
  - 설명:
    - 미국장 시작 전 오늘의 추천 후보를 알려드립니다.
  - 거부 상태 문구:
    - 브라우저 알림 권한이 꺼져 있습니다. 브라우저 설정에서 다시 허용할 수 있습니다.
- 계정:
  - 로그아웃
  - 계정 삭제
- 면책:
  - 투자 참고용 정보이며 투자 자문이 아닙니다.

### 7. 푸시 딥링크 랜딩 화면

목적: 아침 브리핑 푸시를 탭한 사용자가 홈 또는 상세로 자연스럽게 도착하게 한다.

상태별 화면:

1. 딥링크 성공
- 문구: 아침 브리핑에서 온 추천 카드입니다.
- 추천 상세 또는 홈 카드 표시

2. 딥링크 실패
- 문구: 연결된 추천을 찾지 못했습니다.
- CTA: 오늘의 추천 카드 보기

3. 미인증
- 문구: 추천 카드를 보려면 로그인이 필요합니다.
- CTA: 로그인하기

4. No Call
- 문구: 오늘은 명확한 추천을 만들지 않았습니다.
- CTA: 관심 종목 수정하기

## 상태 UX

다음 상태를 반드시 구현해줘.

### Loading

- skeleton card 사용
- 추천 카드, 상세, 이력 리스트 각각 skeleton 표시
- 긴 spinner만 단독으로 보여주지 말 것

### No Call

문구:

- 오늘은 명확한 추천을 만들지 않았습니다.
- 가격 데이터와 뉴스 신호가 충분히 정렬되지 않아 무리한 판단을 피했습니다.
- 관심 종목을 조정하거나 내일 아침 브리핑을 확인하세요.

CTA:

- 관심 종목 수정
- 다시 확인

### Empty

이력 없음:

- 아직 쌓인 추천 이력이 없습니다.
- 추천 카드가 평가되면 성공과 실패 기록이 여기에 표시됩니다.

성과 부족:

- 데이터 축적 중
- 이 종목은 아직 충분한 평가 기록이 없습니다.

### Error

기술 오류 원문, HTTP 5xx, stack trace를 사용자에게 보여주지 말 것.

문구:

- 잠시 후 다시 확인해주세요.
- 외부 데이터 응답이 지연되고 있습니다.

CTA:

- 다시 시도
- 홈으로 이동

## 이벤트 추적 표현

실제 PostHog 연동은 필요 없지만, 개발 확인용으로 화면 우측 하단 또는 설정 내 작은 Debug Panel을 만들어 최근 이벤트를 표시해줘.

이벤트 예시:

- home_view
- rec_card_impression
- rec_card_click
- rec_detail_view
- price_copy
- broker_redirect
- confidence_view
- confidence_change
- performance_card_view
- reason_expand
- push_open
- deeplink_success
- deeplink_fail

사용자가 버튼을 누르거나 화면을 이동할 때 Debug Panel에 최근 이벤트가 쌓이게 해줘.

## 법적/신뢰 카피

다음 문구는 추천 카드와 상세 화면 하단에 항상 보여줘.

- 투자 참고용 정보이며 투자 자문이 아닙니다.
- 실제 투자 결정과 책임은 사용자에게 있습니다.

톤앤매너:

- 과장 금지
- "무조건 오른다", "확실한 수익" 같은 표현 금지
- AI가 정답을 준다는 느낌보다 사용자가 판단할 수 있게 돕는 표현 사용

## 핵심 인터랙션

다음 인터랙션은 동작해야 한다.

1. 로그인 버튼 클릭 -> 온보딩 또는 홈 이동
2. 온보딩 chip 선택/해제
3. 3개 초과 선택 차단
4. 0개 선택 시 CTA disabled
5. Confidence Score 변경
6. Confidence 변경 시 카드 값 즉시 변경
7. 가격 복사 toast
8. 상세 보기 이동
9. 이력 필터 변경
10. 설정에서 관심 종목 수정
11. 푸시 switch on/off
12. No Call 상태 확인
13. Empty 상태 확인
14. Error 상태 확인
15. Debug Panel 이벤트 기록

## 라우팅 구조

프로토타입 안에서 다음 라우트 또는 화면 상태를 구분해줘.

- `/login`
- `/onboarding`
- `/`
- `/recommendations/:recId`
- `/archive`
- `/settings`
- `/push-landing`
- `/state/no-call`
- `/state/empty`
- `/state/error`

실제 URL 라우팅이 어렵다면 앱 내부 state로 화면을 전환해도 된다.

## 완료 기준

구현 결과는 다음을 만족해야 한다.

- 홈에서 추천 카드가 최대 3개만 보임
- 각 추천 카드에 ticker, direction, price, holdDays, confidenceScore, reasonLine이 모두 보임
- Confidence Score가 단순 badge가 아니라 직접 조작 가능한 segmented control임
- Confidence Score 변경 시 카드 값이 바뀜
- 추천 상세 메인 폴드에 차트가 없음
- 성과 이력에 성공과 실패가 모두 보임
- No Call, Loading, Empty, Error 상태가 각각 확인 가능함
- 투자 자문이 아니라는 면책 문구가 추천 카드와 상세에 고정 노출됨
- 모바일과 데스크톱 모두 깨지지 않음
- 사용자가 3분 안에 오늘의 후보, 가격, 기간, 이유를 이해할 수 있는 정보 위계로 구성됨
